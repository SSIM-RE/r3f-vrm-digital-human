## 1. 🎯 整体概念

想象你在建造一个**语音助手**，就像小爱同学或 Siri。

你需要以下几个部分：

| 部分 | 作用 | 比作 |
|------|------|------|
| **麦克风** | 听到用户说话 | 耳朵 |
| **大脑** | 理解用户说了什么 | 大脑 |
| **嘴巴** | 说话回复用户 | 嘴巴 |
| **身体** | 做动作、表情 | 身体 |

我们的项目就是把这些组合起来：

```
麦克风(浏览器) → 大脑(AI) → 嘴巴(语音) + 身体(VRM)
```

### 2.2 语音录制 (useAudioRecorder.js)

**这是什么？**
- 浏览器的麦克风 API
- 负责：打开麦克风 → 录制声音 → 停止录音 → 生成音频文件

**核心代码逻辑：**
```javascript
// 1. 请求麦克风权限
navigator.mediaDevices.getUserMedia({ audio: true })

// 2. 开始录音
mediaRecorder.start()

// 3. 收集音频数据
mediaRecorder.ondataavailable = (event) => {
  audioChunks.push(event.data)
}

// 4. 停止录音，生成文件
new Blob(audioChunks, { type: 'audio/webm' })
```

### 2.3 Whisper 转写服务 (localhost:5000)

**这是什么？**
- 一个把**声音变成文字**的服务
- 类似微信的"语音转文字"功能

**工作原理：**
```
用户说话 (音频) → Whisper API → "你好呀" (文字)
```

**技术实现：**
- Python + faster-whisper 库
- HTTP 接口：`POST /transcribe`

**请求示例：**
```bash
POST http://localhost:5000/transcribe
Content-Type: multipart/form-data

[音频文件]  →  返回  →  {"text": "你好呀"}
```

### 2.4 OpenClaw Gateway (localhost:18789)

**这是什么？**
- 整个系统的大脑和指挥中心
- 接收用户消息 → 调用 AI 模型 → 返回回复

**两大功能：**

#### A. HTTP API - 发送消息
```bash
POST http://localhost:18789/v1/responses
Authorization: Bearer <token>
Content-Type: application/json

{"model": "openclaw:main", "input": "你好呀"}
    ↓
返回 AI 的回复
```

#### B. WebSocket - 实时事件
- 保持长连接，实时推送消息
- 发送心跳，证明自己"还活着"
- 流式推送 AI 的回复

## 4. 📝 代码详解

### 4.1 UI.jsx - 主逻辑文件

文件位置：`src/components/UI.jsx`

#### 第一部分：WebSocket 连接

```javascript
// 连接到 OpenClaw Gateway
const ws = new WebSocket('ws://127.0.0.1:18789/api');

// 连接成功后的认证
ws.onopen = () => {
  ws.send(JSON.stringify({
    type: "req",
    id: "connect",
    method: "connect",
    params: {
      minProtocol: 3,
      maxProtocol: 3,
      client: {
        id: "webchat",
        version: "1.0.0",
        platform: "web",
        mode: "webchat"
      },
      role: "operator",
      auth: {
        token: "bbdc90f999420b51a92a6526a44087037c8aa8e529b52be8"
      },
      locale: "zh-CN",
      userAgent: "Mico-VRM/1.0.0"
    }
  }));
};
```

**解释：**
- WebSocket 就像打电话，是持续连接
- 刚连接时需要"自我介绍"（认证）
- 告诉 Gateway：我是谁、我用什么协议

---

#### 第二部分：接收录音处理

```javascript
useEffect(() => {
  if (audioBlob) {
    // audioBlob 就是录制好的音频文件
    
    // 1. 发送到 Whisper 转写
    const formData = new FormData();
    formData.append('audio', audioBlob, 'recording.webm');
    
    fetch('http://localhost:5000/transcribe', {
      method: 'POST',
      body: formData
    })
    .then(res => res.json())
    .then(data => {
      // 2. 收到转写结果
      const text = data.text;  // 例如："你好呀"
      
      // 3. 发送给 OpenClaw AI
      sendToAI(text);
    });
  }
}, [audioBlob]);

// 发送消息给 AI 的函数
function sendToAI(text) {
  fetch('/v1/responses', {   // Vite 代理到 Gateway
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer bbdc90f999420b51a92a6526a44087037c8aa8e529b52be8'
    },
    body: JSON.stringify({
      model: "openclaw:main",
      input: text
    })
  })
  .then(res => res.json())
  .then(data => {
    console.log('AI 回复:', data);
  });
}
```

---

#### 第三部分：接收 AI 回复

```javascript
ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  
  // 处理 AI 事件
  if (message.type === 'event' && message.event === 'agent') {
    // 流式输出
    console.log('🤖 AI 正在回复:', message.payload);
  }
  
  if (message.type === 'event' && message.event === 'chat') {
    // 最终消息
    const reply = message.payload.message.content;
    console.log('💬 AI 最终回复:', reply);
  }
};
```

### 4.2 vite.config.js - 代理配置

文件位置：`vite.config.js`

```javascript
export default defineConfig({
  server: {
    port: 5173,
    // 关键：代理配置
    proxy: {
      '/v1/responses': {
        // 请求 /v1/responses 时
        target: 'http://127.0.0.1:18789',
        // 转发到 Gateway
        changeOrigin: true
      }
    }
  }
});
```

**为什么需要代理？**
- 浏览器直接请求 `localhost:18789` 会遇到 CORS 错误
- 通过 Vite 代理可以绕过这个问题

### 4.3 useAudioRecorder.js - 录音功能

文件位置：`src/hooks/useAudioRecorder.js`

```javascript
// 开始录音
function startRecording() {
  // 1. 获取麦克风
  navigator.mediaDevices.getUserMedia({ audio: true })
    .then(stream => {
      // 2. 创建录音机
      mediaRecorder = new MediaRecorder(stream);
      
      // 3. 收集数据
      mediaRecorder.ondataavailable = (event) => {
        audioChunks.push(event.data);
      };
      
      // 4. 录音完成时
      mediaRecorder.onstop = () => {
        // 5. 生成 Blob 文件
        audioBlob = new Blob(audioChunks, { 
          type: 'audio/webm'  // 音频格式
        });
        // 6. 通知父组件
        onStop(audioBlob);
      };
      
      // 7. 开始录音
      mediaRecorder.start();
      
      // 8. 设置静音检测
      startSilenceDetection();
    });
}

// 静音检测
function startSilenceDetection() {
  // 持续检查音量
  audioContext.onaudioend = () => {
    // 800ms 没有声音 → 自动停止
    stopRecording();
  };
}
```

```powershell
openclaw gateway status
```

3. 查看 Gateway 日志
```powershell
openclaw logs --lines 50
```

## 6. 💡 开发经验总结

### 6.1 踩过的坑

| 序号 | 问题现象 | 根本原因 | 解决方法 |
|:---:|----------|----------|----------|
| 1 | WebSocket 连接失败 | 协议版本用错 (v1) | 使用 **v3** 协议 |
| 2 | 参数验证失败 | 缺少必需字段 | 添加 `role`, `mode`, `locale` |
| 3 | Device identity 错误 | 空字符串不被接受 | 使用非空字符串 |
| 4 | 缺少 operator.write 权限 | Token 模式不识别 scopes | 改用 **HTTP API** 发送消息 |
| 5 | CORS 跨域错误 | 浏览器不能直接请求 18789 | 用 **Vite 代理** |
| 6 | 400 Bad Request | 请求格式错误 | 使用 `input` 不是 `messages` |

### 6.2 核心教训：权限体系

```
❌ 最初以为：
WebSocket 连接 = 就可以发消息

✅ 实际上：
WebSocket 连接 (token认证) → 只有接收权限
                    → 发送消息需要更高权限
                    → 这个权限在 token 模式下不生效！
```

**最终方案：**
- 📥 **WebSocket** - 只用于接收事件（心跳、AI回复）
- 📤 **HTTP API** - 用于发送消息 (`/v1/responses`)
