# VRM 虚拟数字人 × OpenClaw

> 这是一个**设计思路分享**项目，展示如何将 VRM 虚拟形象与 OpenClaw Agent 结合。

---

## 项目起源

我想做一个可以对话的虚拟数字人，需要：
- 能说话（有 TTS）
- 有表情
- 有动作
- 能理解用户意图

在这个过程中，我探索了一套实现思路，现在分享出来。

---

## 核心设计

### 整体思路

```
OpenClaw Agent (大脑)
       │
       │ 返回 {text, emotion, action}
       ▼
┌──────────────────────────────────────┐
│         前端 (React + R3F)           │
│  • 解析 Agent 返回                   │
│  • TTS 语音合成 + 口型同步           │
│  • 控制 VRM 表情和动作               │
└──────────────────────────────────────┘
```

### 关键设计点

**1. 多模态输出协调**
- Agent 返回结构化数据：`{voiceText, emotion, actions}`
- 前端按顺序触发：先 TTS，播放时同时执行动作和表情

**2. VRM 表情控制**
- VRM 的表情是**累加**的，不需要手动计算混合
- 直接 `setValue()` 即可

**3. TTS 口型同步**
- 用音量驱动嘴巴开合
- 用 `isSpeaking` 判断状态，不是口型值（静音 ≠ 停止）

**4. 动作生成**
- 预设动画（Mixamo）优先，响应快
- AI 生成（DiP）作为增强，更自然

## 模块说明

| 模块 | 职责 | 关键点 |
|------|------|--------|
| VRMAvatar | VRM 渲染、表情、动画 | 版本兼容、骨骼控制 |
| useVRMControl | 状态管理 | 表情渐变、动作队列 |
| useTTS | 语音合成 | 频谱分析、口型同步 |
| useDipMotion | AI 动作 | 骨骼映射、帧应用 |
| useAudioRecorder | 录音 | VAD 自动截断 |

---

## 经验总结

1. **VRM 表情是累加的** - 直接 setValue，不用计算混合
2. **用 isSpeaking 判断 TTS 状态** - 不是口型值
3. **动作优先用预设** - DiP 作为增强
4. **骨骼映射要翻转轴** - HML 和 VRM 坐标系不同

## 为什么不是复刻项目

- 外部依赖多（GPT-SoVITS、DiP 模型、OpenClaw）
- 需要 GPU 运行 DiP
- 部分逻辑和 OpenClaw 深度绑定

这个项目更重要的是**思路分享**：遇到类似需求时，可以参考这个架构。
