# VRM 虚拟数字人 × OpenClaw

> 这是一个**设计思路分享**项目，展示如何将 VRM 虚拟形象与 OpenClaw Agent 结合。

---

## 实现效果

这个项目实现了什么：

```
用户说话 → 虚拟形象回应（语音+表情+动作）
```

1. **能对话** - 用户语音/文本输入，OpenClaw Agent 理解意图后返回结构化回复
2. **会说话** - TTS 语音合成，虚拟形象能开口
3. **有表情** - 根据 Agent 返回的情绪（开心、悲伤、惊讶等）改变面部表情
4. **有动作** - 根据 Agent 返回的动作指令播放对应动画
5. **口型同步** - 说话时嘴巴会跟着动

```
│  • VRM 动作控制                          │
└─────────────────────────────────────────┘
```

## 设计思路

### 1. 多模态输出协调

问题：语音、表情、动作需要同时响应，如何协调？

解决：Agent 返回结构化数据 `{voiceText, emotion, actions}`，前端按顺序触发：
- 先发起 TTS
- 播放时同时执行表情和动作

### 2. 动作实现方式

虚拟形象的动作有两种实现方式：

| 方式 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **预设动画** | 使用 Mixamo 等预制动画 | 快速、稳定 | 动作有限 |
| **DiP 生成** | AI 生成动作 | 自然、灵活 | 需要 GPU |

实际项目中可以结合使用：常用动作用预设，特殊动作用 AI 生成。

### 3. 状态判断

问题：怎么知道 TTS 什么时候播放完？

解决：用明确的布尔值状态 `isSpeaking` 判断，而不是依赖计算值（口型大小），避免误判。

## 模块结构

| 模块 | 职责 |
|------|------|
| VRMAvatar | VRM 渲染、表情控制、动画播放 |
| useVRMControl | 状态管理（表情渐变、动作队列） |
| useTTS | 语音合成、口型同步 |
| useDipMotion | AI 动作生成（可选） |
| useAudioRecorder | 录音（带自动截断） |

## 为什么是思路分享

- 外部依赖多（GPT-SoVITS、DiP 模型、OpenClaw）
- 需要 GPU 运行 AI 动作
- 部分逻辑和 OpenClaw 深度绑定

更重要的是**架构思路**：遇到类似需求时，可以参考这个设计。
