# VRM 虚拟数字人 × OpenClaw

> 这是一个**设计思路分享**项目，展示如何将 VRM 虚拟形象与 OpenClaw Agent 结合。

---

## 项目起源

我想做一个可以对话的虚拟数字人，需要：
- 能说话（有 TTS）
- 有表情
- 有动作
- 能理解用户意图

在这个过程中，我探索了一套实现思路，现在分享出来。

---

## 整体架构

```
用户输入 (语音/文本)
       │
       ▼
┌─────────────────────────────┐
│      OpenClaw Agent         │
│   (理解意图，返回指令)       │
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│              前端处理                   │
│  • TTS 语音合成 + 口型同步               │
│  • VRM 表情控制                          │
│  • VRM 动作控制                          │
└─────────────────────────────────────────┘
```

---

## 核心设计思路

### 1. 多模态输出协调

Agent 返回结构化数据，前端解析后协调触发：

```
Agent 返回 {voiceText, emotion, actions}
       │
       ▼
先发起 TTS → 播放时同时执行表情 + 动作
```

### 2. 动作优先用预设

- **预设动画**（Mixamo）：响应快、稳定
- **AI 生成**（DiP）：更自然、灵活，但需要 GPU

### 3. 状态管理

用 `isSpeaking` 这种明确状态判断，而不是依赖计算值。

## 模块说明

| 模块 | 职责 |
|------|------|
| VRMAvatar | VRM 渲染、表情控制、动画播放 |
| useVRMControl | 状态管理（表情渐变、动作队列） |
| useTTS | 语音合成、口型同步 |
| useDipMotion | AI 动作生成（可选） |
| useAudioRecorder | 录音（带自动截断） |

---

## 为什么是思路分享

- 外部依赖多（GPT-SoVITS、DiP 模型、OpenClaw）
- 需要 GPU 运行 AI 动作
- 部分逻辑和 OpenClaw 深度绑定

更重要的是**架构思路**：遇到类似需求时，可以参考这个设计。
