# 技术决策

## 决策记录

### [DEC-001] 使用 Zustand 进行状态管理
- 日期: 2026-02-24
- 方案对比:
  - Zustand: API 简洁，性能优秀，无需 Provider 包裹
  - Redux: 生态成熟，但学习曲线陡，配置复杂
  - Context API: 简单直接，但性能优化需要手动处理
- 最终选择: Zustand
- 原因:
  - 项目规模适中，Zustand 足够使用
  - API 简洁，减少样板代码
  - 性能优秀，支持异步操作
  - 与 React 19 兼容性好
- 影响: 整个应用的状态管理都基于 Zustand

### [DEC-002] 使用 Vite 作为构建工具
- 日期: 2026-02-24
- 方案对比:
  - Vite: 启动快，HMR 优秀，配置简单
  - Webpack: 功能强大，但配置复杂，启动慢
- 最终选择: Vite
- 原因:
  - 开发体验好，热更新快
  - 开箱即用，零配置支持 React
  - 生态活跃，插件丰富
- 影响: 整个项目的构建流程基于 Vite

### [DEC-003] 使用 GPT-SoVITS 作为 TTS 服务
- 日期: 2026-03-02
- 方案对比:
  - GPT-SoVITS: 开源免费，效果好，本地部署
  - 在线 TTS API: 需要付费，可能有延迟
  - Web Speech API: 浏览器内置，但效果一般
- 最终选择: GPT-SoVITS
- 原因:
  - 免费开源，可本地部署
  - 语音效果好，接近真人
  - 支持中文
  - 可自定义声音
- 影响: 需要 GPT-SoVITS 服务在本地运行
